## Unleash Configuration[​](#unleash-setup "Direct link to Unleash Configuration")

<Info>
  Before you begin, ensure that you have a strategy in place to prevent [being locked out of all admin accounts](/troubleshooting#got-locked-out-of-an-admin-account-after-configuring-scim).
</Info>

### Step 1: Navigate to Provisioning configuration[​](#unleash-setup-step-1 "Direct link to Step 1: Navigate to Provisioning configuration")

First you'll need to log in to Unleash as an admin user. Navigate to the Single Sign-On section and select the "SCIM" tab. The SCIM API URL will be shown in this section, you'll need this to configure Okta later.

![Navigate to the SCIM Config](../images/how-to/assets/images/scim-config-1-40b9ecac4d5adf29e791ada889090f43.png)

### Step 2: Enable Provisioning[​](#unleash-setup-step-2 "Direct link to Step 2: Enable Provisioning")

Enable SCIM by turning on the toggle and keep the token Unleash provides you for the Okta setup below.

![Enable the SCIM toggle](../images/how-to/assets/images/scim-config-2-8507e8b0c509a882b98c8f012134253f.png)

## Okta Configuration[​](#okta-setup "Direct link to Okta Configuration")

### Step 1: Create an Application in Okta[​](#okta-setup-step-1 "Direct link to Step 1: Create an Application in Okta")

<Info>
  If you already have SAML SSO configured for Unleash in Okta you can skip to the [next step](/how-to/how-to-setup-provisioning-with-okta#okta-setup-step-2). If you're planning on using [SAML for Unleash](/how-to/how-to-add-sso-saml), do that first and skip to the next step. Note that if you're using OIDC SSO in Okta you still need to do this step.

  This step will create an empty Sign-On Application that will only be used for SCIM.
</Info>

**1) Navigate to "Admin -> Applications" and click the "Create App Integration" button.**

![Navigate to Create Application](../images/how-to/assets/images/scim-okta-config-1-71a53f69cdd3dc010e1430a13a06c4c0.png)

**2) Select SWA - Secure Web Authentication**

![Select Secure Web Application](../images/how-to/assets/images/scim-okta-config-2-aeea7f74a01f032593855ae3d29077cb.png)

**3) Fill in your App Name and App's login page URL**

![Setup Application Properties](../images/how-to/assets/images/scim-okta-config-3-8c98d0b4058215c5d923c06653308f12.png)

### Step 2: Enable Provisioning in your Okta Application[​](#okta-setup-step-2 "Direct link to Step 2: Enable Provisioning in your Okta Application")

<Info>
  If you already have a SAML application setup for Unleash you'll be modifying that application in this step.
</Info>

**Enable SCIM provisioning and save.**

![Enable SCIM](../images/how-to/assets/images/scim-okta-config-5-b68c8d4003c0ab7649693460dc812d46.png)

### Step 3: Connect Unleash[​](#okta-setup-step-3 "Direct link to Step 3: Connect Unleash")

**1) Navigate to the Provisioning tab**

**2) Set the Unleash SCIM URL**

This is provided by the Unleash UI in the [configuring Unleash](/how-to/how-to-setup-provisioning-with-okta#unleash-setup-step-1) section.

**2) Set email as the unique identifier**

**3) Configure actions**

Turn on "Push New Users", "Push Groups" and "Push Profile Updates".

**4) Set authentication mode to "HTTP Header"**

**5) Add your SCIM token**

This was provided by the Unleash UI in the [configuring Unleash](/how-to/how-to-setup-provisioning-with-okta#unleash-setup-step-2) section.

![Connect Unleash](../images/how-to/assets/images/scim-okta-config-5-b68c8d4003c0ab7649693460dc812d46.png)

### Step 4: Configure Okta Provisioning[​](#okta-setup-step-4 "Direct link to Step 4: Configure Okta Provisioning")

Navigate to the "To App" tab. Turn on "Create Users", "Update User Attributes" and "Deactivate Users". Save your configuration.

![Configure Okta Provisioning](../images/how-to/assets/images/scim-okta-config-6-56cc04dccba7a6476f04d8f390a98487.png)

### Step 5: Configure Provisioning Properties[​](#okta-setup-step-5 "Direct link to Step 5: Configure Provisioning Properties")

**1) Set email**

Set the email field to map to your login property. This is important and ensures that your SSO integration continues to work.

**2) Remove unneeded properties**

You should remove all unnecessary properties. This ensures that Okta will reach a steady state when synchronizing. The properties that you must retain are:

* Username
* Given name
* Family name
* Email
* Primary email type
* Display name

![Configure Provisioning Attributes](../images/how-to/assets/images/scim-okta-config-7-53876e894a714bf3b28a48dfc173ba76.png)
